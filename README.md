1. 了解 GitHub 基本使用方式；
2. 初步掌握利用 HTML 进行网页的设计方法；
3. 掌握 Web 项目中相对路径和绝对路径概念。

## 实验内容
本实验通过编写 HTML 静态页面，并将其部署到 GitHub Pages 上，实现了一个简单的个人博客。
页面中包含了：
- **绝对路径图片测试**：直接引用互联网上的图片资源（`http://...`）。
- **相对路径图片测试**：引用项目内部 `image` 目录下的图片资源（`image/1.png`）。

## 思考题解答

**1. 在页面中是否可以引入本地绝对路径图片，如：`d:/images/1.png`？及原因**
> **不可以**。当网页发布到互联网后，其他用户通过浏览器访问时，浏览器会尝试在**用户自己**的电脑的 D 盘去寻找该路径。因为用户的电脑上并没有这张图片，所以图片会显示破损。Web 页面中的绝对路径必须是基于 HTTP/HTTPS 协议的网络 URL，或者是相对于 Web 服务器根目录的路径，绝不能是本地文件系统的物理路径。

**2. 由于 github.io 为静态页面托管，如何为自己的博客添加评论功能？**
> 因为 GitHub Pages 只能托管静态文件，不支持运行后端代码和数据库，所以无法自己实现评论的存储。解决方法是借助**第三方的评论插件系统**：
> - **Gitalk / Giscus / utterances**：利用 GitHub 的 Issue 或 Discussions 功能来存储评论，非常适合托管在 GitHub Pages 上的博客。
> - **Valine**：基于 LeanCloud 的无后端评论系统。
> - **Disqus / 畅言**：传统的第三方社会化评论系统（只需在 HTML 中引入一段 JS 代码即可）。
